## 1. プロジェクト概要

### 1.1 目的

MT5(MetaTrader 5)とPythonの通信により、マルチタイムフレーム分析・複数銘柄監視・通貨間強弱を統合した、**視認性最優先のローカルWebダッシュボード**を構築する。

### 1.2 役割分担(厳密に守る)

```
MT5      → 注文端末、チャート描画、TL/SR描画
Python   → 分析エンジン、計算、データ集約
Browser  → 視認性最優先の統合ダッシュボード(Dash)
```

### 1.3 トレードスタイル前提

- 完全裁量(自動売買・自動発注は実装しない)
- マルチタイムフレーム・トレンドフォロー
- 2-3日跨ぎ保有
- 構造ベースアプローチ(TL/SR優先、指標は背景情報)
- 主戦時間帯:21:00-25:00 JST(London-NY overlap)

│  MT5         │←──→│  Python          │←──→│  Browser     │
│  - 注文端末  │IPC │  - 分析エンジン  │ WS │  - Dash表示  │

└──────┬───────┘    └────────┬─────────┘    └──────────────┘
       ↑                     ↑
       │                     │
   MQL5 EA            Forex Factory XML

### 2.1 プロセス構成

| プロセス | 役割 |
|---------|------|
| MT5 Terminal | 常時起動・ログイン |
| LineExporter EA | MT5チャート常駐、TL/SR JSON出力 |
| Python Backend | 分析・Dashサーバー |
| Browser | localhost:8050 表示 |

### 2.2 通信フロー

```
1. MT5 ←IPC→ Python (MetaTrader5 package)
2. MT5 EA → JSONファイル → Python (watchdog監視)
3. Python ←WebSocket→ Browser
4. Forex Factory → HTTP → Python
```

## 6. インジケーター仕様(厳密)

### 6.1 EMA(各TF 1本のみ)

| TF | 期間 | 用途 |
|----|------|------|
| D1 | 200 | 大局方向許可 |
| H4 | 50 | スイング方向 |
| H1 | 20 | 短期方向 |
| M15 | 13 | 執行文脈 |

- 種別:EMA(Exponential Moving Average)
- ソース:Close
- 表示:価格との位置関係のみ(線の交差をシグナル化しない)

### 6.2 ADX(14)

- 表示TF:D1とH4
- 閾値:25未満=レンジ、25-30=トレンド、30+=強トレンド
- GOLDの場合:30以上を採用閾値
- DI+/DI-表示:オプション(画面圧迫時は省略)

### 6.3 RSI(14)

- 表示TF:H1とM15
- 過熱判定:>70警戒、<30警戒
- ダイバージェンス検出機能(M15のみ)
- 単独シグナル化禁止

### 6.4 ATR(14)

- 全TF計算、画面表示はH4のみ
- 用途:SL距離算出、構造接近判定の基準
- 計算式:標準ATR(Wilder法)

### 6.5 完全不採用(絶対実装しない)

```
× 一目均衡表(雲含む全要素)
× MACD
× Stochastic
× CCI
× Bollinger Bands
× フィボナッチ自動描画
× オシレーター系全般
× MA・雲タッチシグナル
× 各種クロスオーバー判定
```

## 8. 構造ベースアプローチ

### 8.1 シグナル発生源の階層

```
一次(絶対信頼):
  - ユーザー描画TL
  - ユーザー描画S/R
  - PDH/PDL/PWH/PWL
  - ラウンドナンバー
  - コンフルエンスゾーン

二次(確認・補強):
  - プライスアクションパターン
  - ATR(SL算出のみ)

三次(背景情報、シグナル化しない):
  - EMA(方向確認のみ)
  - ADX(強度のみ)
  - RSI(過熱のみ)

不採用(完全に無視):
  × MA反発・MAクロス
  × 雲タッチ・雲ブレイク
  × MACDクロス
  × RSI 30/70単独
```

### 8.2 重要原則

- **指標は方向確認の補助、エントリー判断には使わない**
- **構造的レベルへの接触+価格行動確認のみがシグナル根拠**

## 9. TL/SR読込仕様

### 9.1 MQL5 EA「LineExporter」要件

**機能**:
- MT5チャート上の描画オブジェクト全走査
- OnChartEvent:線の追加・移動・削除を即時検知
- OnTimer 5秒:既存ライン整合性確認
- JSON形式で出力

**出力先**:
```
[MT5_DataFolder]/MQL5/Files/Common/lines_{symbol}.json
```

**読取対象**:

| 種別 | OBJ定数 | 取得属性 |
|------|---------|---------|
| トレンドライン | OBJ_TREND | 2点座標、傾き、レイ延長 |
| 水平線 | OBJ_HLINE | 価格 |
| 長方形 | OBJ_RECTANGLE | 4点座標 |
| チャネル | OBJ_CHANNEL | 平行2線 |
| フィボ | OBJ_FIBO | 起点・終点、レベル |
| テキスト | OBJ_TEXT | 注釈 |

### 9.2 JSON出力フォーマット

```json
{
  "symbol": "XAUUSD",
  "updated_at": "2026-05-18T23:45:12+09:00",
  "lines": {
    "horizontal": [
      {
        "name": "R1_strong_D1",
        "price": 3265.00,
        "color": "red"
      }
    ],
    "trendlines": [
      {
        "name": "TL_up_D1",
        "point1": ["2026-05-10T00:00", 3180.50],
        "point2": ["2026-05-15T00:00", 3220.30],
        "ray_right": true,
        "current_value": 3242.20,
        "slope_per_day": 7.96
      }
    ],
    "rectangles": [],
    "fibonacci": []
  }
}
```

### 9.3 命名規則(ユーザー側で守る)

| 接頭辞 | 意味 |
|-------|------|
| `R_` `R1_` `R2_` | レジスタンス |
| `S_` `S1_` `S2_` | サポート |
| `TL_up_` | 上昇TL |
| `TL_dn_` | 下降TL |
| `zone_supply` | 売り需要ゾーン |
| `zone_demand` | 買い需要ゾーン |
| `_strong` `_major` `_weak` | 重要度修飾子 |
| `_D1` `_H4` `_H1` | 描画基準TF修飾子 |

Pythonは接頭辞・修飾子から自動カテゴリ分類する。

### 9.4 反映速度要件

| 操作 | 反映時間 |
|------|---------|
| 新規描画 | 1秒以内 |
| 移動 | 1秒以内 |
| 削除 | 1秒以内 |

## 11. プライスアクション検出

### 11.1 対象TF

**M15のみ**(他TFは検出しない)

### 11.2 検出パターン

| パターン | 数学的定義 |
|---------|-----------|
| **ピンバー(ブル)** | 下ヒゲ ≥ 実体×2、上ヒゲ < 実体×0.3 |
| **ピンバー(ベア)** | 上ヒゲ ≥ 実体×2、下ヒゲ < 実体×0.3 |
| **強気包み足** | 当該足の陽線実体が前足の陰線実体を完全包含 |
| **弱気包み足** | 当該足の陰線実体が前足の陽線実体を完全包含 |
| **インサイドバー** | 当該足の高安が前足範囲内 |
| **インサイドバーブレイク** | インサイドバー直後のN本目で方向ブレイク |
| **3バー反転** | 3本連続の反転構造 |

### 11.3 表示条件

- 構造的レベル接触時のみハイライト
- 単体検出時は内部記録のみ、UI表示しない

## 12. 通貨強弱計算

### 12.1 対象通貨

```
XAU(GOLD), USD, EUR, GBP, JPY, AUD, CHF, NZD
※XAUは参考表示、計算は7法定通貨のみ
```

### 12.2 計算用ペア(背景処理)

```
EURUSD, GBPUSD, AUDUSD, NZDUSD,
USDJPY, USDCHF, USDCAD,
EURJPY, GBPJPY, AUDJPY, NZDJPY, CADJPY, CHFJPY,
EURGBP, EURAUD, EURNZD, EURCHF, EURCAD,
GBPAUD, GBPNZD, GBPCHF, GBPCAD,
AUDNZD, AUDCHF, AUDCAD,
NZDCHF, NZDCAD,
CADCHF
```

### 12.3 計算方法

```
1. 各ペアの一定期間変化率を計算
2. 各通貨について、その通貨が含まれる全ペアの変化率を合成
   - 通貨がベース通貨の場合:+変化率
   - 通貨がクォート通貨の場合:-変化率
3. 0-100スケールに正規化
```

### 12.4 時間窓

```
H1(直近1時間)
H4(直近4時間)
D1(直近1日)
W1(直近1週)
```

切替可能なUI実装。

### 12.5 更新頻度

**30秒間隔**

### 12.6 ペア優位度

```
各取引ペアについて自動算出:
  例:GBPJPY = GBP強度 - JPY強度
  
判定:
  Δ +30以上: STRONG BUY
  Δ +10〜+30: BUY優位
  Δ -10〜+10: NEUTRAL
  Δ -30〜-10: SELL優位
  Δ -30以下: STRONG SELL
```

## 18. 起動・運用

### 18.1 自動起動シーケンス

```
1. Windows起動
2. MT5自動起動(MT5のスタートアップオプション利用)
3. MT5自動ログイン(保存済認証情報)
4. LineExporter EA自動起動(チャートテンプレート利用)
5. Python起動(タスクスケジューラ or バッチ)
   - MT5起動から30秒後を推奨
6. Dashサーバー起動(localhost:8050)
7. ブラウザ自動起動
   - kioskモード or F11全画面
8. 監視運用開始
```

### 18.2 起動スクリプト

`scripts/start_dashboard.bat`:
```batch
@echo off
timeout /t 30 /nobreak
cd /d %~dp0..
python main.py
```

### 18.3 24時間稼働設定

```
Windows電源設定:
  電源プラン:高パフォーマンス
  スリープ:無効
  ハイバネーション:無効
  ディスプレイOFF:30分OK
  システムスリープ:絶対無効

ネットワーク:
  有線推奨
  Wi-Fi切断時の自動再接続設定
```

### 18.4 異常時対応

| 状況 | 対応 |
|------|------|
| MT5接続切断 | Python側で5秒間隔再接続試行 |
| Python落ち | タスクスケジューラで自動再起動 |
| EA停止 | MT5側で監視(WARNING表示) |
| メモリリーク | 週次再起動推奨 |
